# Paquete de prueba para junta con Tony

Fecha sugerida de uso: martes 16 de junio de 2026.

Este paquete sirve para mostrar LexContratos con datos ficticios, sin usar documentos reales de clientes, empresas del grupo ni abogados. La idea es que Tony pueda ver el flujo normal de trabajo: seleccionar un formato, guardar el documento, aplicar membrete, cargar documentos por parte, extraer datos, integrar faltantes, revisar y exportar.

## Qué contiene

- `01_GUION_JUNTA_TONY.md`: guion para conducir la sesión.
- `02_CHECKLIST_DE_PRUEBA.md`: lista de validación para Tony y su equipo.
- `03_FORMATO_OBSERVACIONES_TONY.md`: formato para levantar comentarios claros.
- `formatos-y-flujo/FLUJO_NORMAL_DE_TRABAJO.md`: paso a paso del flujo dentro de la app.
- `documentos-de-prueba/`: documentos ficticios para probar extracción.
- `membretes-demo/`: membretes y logos ficticios para enseñar cómo se vería la función.

## Flujo recomendado para la demo

1. Entrar a LexContratos con una cuenta de prueba activa.
2. Abrir `Mis Documentos` y mostrar la estructura general.
3. Ir a `Formatos del sistema` y seleccionar un formato.
4. Usar `Guardar como` para decidir dónde guardar el documento.
5. Seleccionar un membrete del catálogo o mostrar cómo se agregaría.
6. Detectar campos editables.
7. Abrir `Datos y documentos`.
8. Cargar documentos ficticios por cada parte.
9. Ejecutar `Extraer e integrar datos`.
10. Revisar qué campos quedaron pendientes.
11. Integrar manualmente los datos faltantes.
12. Ejecutar `Revisión crítica IA`.
13. Exportar a Word o PDF para firma.
14. Volver a `Mis Documentos` para localizar el documento guardado.

## Importante para la junta

Usar solamente los archivos de este paquete. Todos los nombres, RFC, domicilios, poderes, correos y sociedades son ficticios.

El objetivo de la reunión no es que todo esté perfecto, sino validar si el flujo ya se entiende, si los botones están en el lugar correcto y si Tony puede imaginar a sus abogados usándolo sin instrucciones excesivas.

# Checklist de prueba para Tony y equipo

Usar este checklist durante o después de la junta. La idea es marcar lo que funciona, lo que no se entiende y lo que falta ajustar.

## Acceso

- [ ] Pude entrar con mi usuario.
- [ ] La pantalla inicial se entiende.
- [ ] El acceso a la app privada se encuentra fácilmente.
- [ ] No vi mensajes técnicos o de desarrollo.

## Formatos del sistema

- [ ] Pude encontrar `Formatos del sistema`.
- [ ] Los formatos se ven como documentos utilizables, no como una lista técnica.
- [ ] Pude seleccionar un formato.
- [ ] El formato elegido se abrió como copia de trabajo.
- [ ] Entiendo que una administradora puede ajustar formatos del sistema.
- [ ] Entiendo que modificar un formato del sistema debe requerir confirmación.

## Guardar como y Mis Documentos

- [ ] Pude usar `Guardar como`.
- [ ] Pude elegir dónde guardar el documento.
- [ ] Pude crear una carpeta.
- [ ] Pude crear una subcarpeta dentro de otra carpeta.
- [ ] Pude renombrar el documento durante el guardado.
- [ ] Pude localizar el documento después.
- [ ] El explorador se siente parecido a una carpeta normal.
- [ ] Pude seleccionar varios elementos con Shift o Cmd/Ctrl.

## Documentos de las partes

- [ ] La ventana de `Datos y documentos` se entiende.
- [ ] Las partes mostradas coinciden con el tipo de contrato.
- [ ] Pude subir documentos del cliente o parte receptora.
- [ ] Pude subir documentos de la contraparte, proveedor o parte obligada.
- [ ] Pude subir más de un documento por parte.
- [ ] Se entiende dónde quedan guardados los documentos soporte.

## Extracción e integración

- [ ] El botón `Extraer e integrar datos` se identifica como acción principal.
- [ ] La extracción tomó datos de los documentos.
- [ ] Los datos extraídos se integraron en el contrato.
- [ ] Los campos no encontrados quedaron como pendientes.
- [ ] No se mezclaron los datos de una parte con la otra.
- [ ] La información extraída se pudo revisar antes de confiar en ella.

## Datos faltantes

- [ ] Pude ver qué datos faltaban.
- [ ] Pude capturar datos manualmente.
- [ ] Pude integrar varios datos faltantes al mismo tiempo.
- [ ] Los datos ya llenados dejaron de aparecer como pendientes.

## Revisión crítica IA

- [ ] La revisión crítica se entiende como apoyo, no como sustituto de revisión legal.
- [ ] Las observaciones son útiles.
- [ ] Las observaciones se pueden aceptar o descartar de forma individual.
- [ ] La app advierte si hay pendientes antes de exportar.

## Membretes

- [ ] Pude ver el catálogo de membretes.
- [ ] Pude seleccionar un membrete.
- [ ] Pude quitar el membrete.
- [ ] El membrete se ve profesional en el documento exportado.
- [ ] La información de domicilio o pie de página se muestra correctamente.

## Exportación

- [ ] Pude exportar a Word.
- [ ] El Word se ve presentable.
- [ ] Pude exportar PDF para firma.
- [ ] El PDF se entiende como versión lista para enviar a firma externa.

# Índice oficial del paquete de prueba

Usar este índice como punto de entrada para la junta con Tony.

## Archivos principales

1. `00_LEEME_PAQUETE_PRUEBA.md`
   Explica qué contiene el paquete y cómo usarlo.

2. `01_GUION_JUNTA_TONY.md`
   Guion para conducir la junta paso a paso.

3. `02_CHECKLIST_DE_PRUEBA.md`
   Checklist para validar acceso, formatos, guardado, documentos, extracción, membretes y exportación.

4. `03_FORMATO_OBSERVACIONES_TONY.md`
   Formato para que Tony y su equipo reporten hallazgos de forma ordenada.

5. `formatos-y-flujo/FLUJO_NORMAL_DE_TRABAJO.md`
   Paso a paso operativo dentro de LexContratos.

## Material de prueba

1. `documentos-de-prueba/cliente-contratante/`
   Documentos ficticios de la parte que contrata, compra, recibe o solicita.

2. `documentos-de-prueba/proveedor-contraparte/`
   Documentos ficticios de la contraparte, proveedor, vendedor o parte obligada.

3. `documentos-de-prueba/correo_servicio_contratado_demo.eml`
   Correo ficticio con importe, vigencia, servicio y fecha de firma.

4. `documentos-de-prueba/datos_generales_partes_demo.csv`
   Tabla ficticia de datos generales para prueba.

5. `membretes-demo/`
   Logos y membretes ficticios para enseñar el flujo de selección.

## Nota

Si aparecen otros archivos dentro de la carpeta, pueden conservarse como material complementario, pero para la junta usar primero los documentos listados arriba.

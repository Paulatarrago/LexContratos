# Guion para junta con Tony

Duración sugerida: 45 a 60 minutos.

## 1. Apertura

Tony, la idea de esta sesión es revisar el flujo real de trabajo con LexContratos: cómo se selecciona un formato, cómo se guarda el documento, cómo se cargan documentos de las partes, cómo se extraen datos, cómo se revisa el contrato y cómo se exporta.

La plataforma todavía está en etapa de ajuste, así que lo que más nos interesa es identificar qué ya funciona de forma natural y qué todavía se siente confuso para el uso diario del equipo.

## 2. Acceso y vista general

Mostrar:

- Acceso con usuario.
- Pantalla principal.
- Botón `Mis Documentos`.
- Botón `Seleccionar formato`.
- Botones del flujo de trabajo.
- Catálogo de membretes.

Mensaje clave:

LexContratos no reemplaza la revisión jurídica. Ayuda a reducir captura repetitiva, organizar documentos y preparar contratos editables para que el usuario revise y decida.

## 3. Seleccionar formato y guardar

Mostrar:

- Entrar a `Formatos del sistema`.
- Seleccionar un formato.
- Usar `Guardar como`.
- Elegir una ubicación en `Mis Documentos`, `Clientes`, `Proveedores`, `Empresas del Grupo`, `Documentos de las partes` u `Otros`.
- Crear una carpeta de prueba.
- Renombrar el documento durante el guardado.

Pregunta para Tony:

¿La lógica de guardar se parece suficientemente a un explorador normal de archivos?

## 4. Membretes

Mostrar:

- Catálogo de membretes.
- Selección de un membrete.
- Resultado visual esperado en el documento.
- Opción de quitar membrete.

Pregunta para Tony:

¿Prefieren que los membretes los cargue una administradora desde la plataforma o que se precarguen inicialmente con archivos que nos compartan?

## 5. Datos y documentos

Mostrar:

- Botón `Datos y documentos`.
- Ventana de carga por cada parte.
- Carga de documentos ficticios.
- Botón `Extraer e integrar datos`.
- Resultado de extracción.

Mensaje clave:

El sistema debe entender los roles según el formato seleccionado. No todo es prestador-cliente; cada contrato debe mostrar las partes correctas.

## 6. Datos faltantes y revisión

Mostrar:

- Campos detectados.
- Campos llenados automáticamente.
- Campos que quedaron pendientes.
- Integración manual de faltantes.
- `Revisión crítica IA`.

Preguntas para Tony:

- ¿Qué tipo de alerta sería más útil antes de exportar?
- ¿Qué errores o inconsistencias les gustaría que la IA marque como prioridad?

## 7. Exportación

Mostrar:

- Exportar Word.
- Exportar PDF para firma.
- Explicar que por ahora la firma se hará fuera de LexContratos, subiendo el PDF o Word a la herramienta de firma que usen.

## 8. Cierre

Solicitar comentarios concretos:

- Qué sí se entendió.
- Qué no se encontró.
- Qué paso se sintió innecesario.
- Qué documento o formato real les gustaría probar después.
- Qué ajuste es indispensable antes de invitar a más usuarios.
